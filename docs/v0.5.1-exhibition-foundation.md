# v0.5.1：精品展厅共用底座

> 状态：已实现并通过本地质量门禁。  
> 范围：为京剧与茶文化馆建立可审计的共用内容结构和参观收尾能力；不把“已拥有 8 件资产”或“已有两位人物”误写成既成事实。

## 1. 交付目标

本版对应《UNESCO 中国非遗精品展厅 MVP》5.0 的第一批共用缺口：

1. 可跳过的 15 秒氛围入口；
2. 资产台账、字段级来源、社区节点和保护叙事的数据结构；
3. 两馆可见的“保护进行时”与“带走”模块；
4. 分享降级与下一展厅路径。

社区节点和“至少 8 件合规资产”是 v0.5.2 / v0.5.3 的内容生产目标。本版先登记现有资产、显示当前缺口，并以 Schema 防止未来未标明来源或权利的资产进入展厅。

## 2. 共用内容模型

每个精品展厅新增一条 `exhibition-foundations.json` 记录：

```ts
type ExhibitionFoundation = {
  exhibitionId: "jingju" | "traditional-tea";
  updatedAt: string;
  atmosphere: { title: string; description: string; durationSeconds: 15; disclosure: string };
  factSources: Array<{ id: string; label: string; sourceId: string }>;
  sources: Array<{ id: string; title: string; url: string }>;
  assets: Array<{ id: string; title: string; type: "image" | "audio" | "panorama"; sourceId: string; rights: string; disclosure?: string }>;
  communityNodes: Array<{ id: string; title: string; kind: "person" | "community" | "institution"; sourceId: string }>;
  safeguard: { title: string; summary: string; actions: string[]; sourceId: string };
  takeaway: { learningCard: string; nextExhibitionId: string; nextLabel: string };
};
```

校验规则：`factSources`、资产、保护叙事与社区节点必须引用现有 `sourceId`；资产必须带权利说明；`communityNodes` 在 v0.5.1 可以为空但 UI 必须显示“内容采集中”，后续版本改为最少 2 项。

## 3. 氛围入口

浏览器会拦截无用户手势的有声自动播放，因此入口不自动发声。用户可在标题覆盖层点击“播放 15 秒概念氛围声”，也可直接“进入展厅”。音色由 Web Audio API 在本地合成，明确标注为概念氛围声，不模拟传承人或历史录音；播放、停止和跳过均为原生按钮。

## 4. 带走与保护

每馆末尾显示：

- 一张可复制的学习卡文案；
- 原生系统分享；不支持时复制当前 Hash 链接并提示；
- 下一精品展厅入口；
- 保护叙事、可行动作和来源；
- 资产与社区节点当前进度（不将筹备状态冒充为完成）。

## 5. 验收

1. 两馆入口均可播放或跳过 15 秒概念氛围声，键盘可操作，且不自动播放。
2. 资产、字段来源、保护叙事均能通过 Zod 校验。
3. 分享失败时复制链接可用，用户获得状态反馈。
4. 两馆均有下一展厅入口、保护模块和内容进度说明。
5. 格式、lint、类型、单测、构建和 E2E 全部通过。
