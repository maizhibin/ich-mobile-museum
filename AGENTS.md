# AGENTS.md

## 项目目标

本仓库用于建设“UNESCO 中国非遗数字博物馆”。当前阶段优先交付可部署到 GitHub Pages 的纯前端 H5，围绕专题馆与精品展厅逐步覆盖中国列入 UNESCO 非物质文化遗产名录、名册的项目。

## 基本规则

- 始终使用简体中文沟通、撰写计划和说明文档。
- 开始实现新技术方案前，优先查阅官方文档和当前最佳实践；不要引用内容农场。
- JS/TS 使用 ESM；React 使用函数组件和 Hooks，禁止新增 class 组件。
- TypeScript 开启严格模式，避免 `any`；数据边界必须通过 Schema 校验。
- 优先具名导入；保持组件小而清晰，按功能组织代码。
- 不在前端保存任何密钥、私人联系方式或受限非遗资料。
- UNESCO 核心项目、国家级对照项目和知识节点必须在数据与界面中明确区分。
- AI 生成内容必须明确标注，并与真人录音、历史资料分开。

## 目录约定

- `src/app`：路由、应用外壳和全局 Provider。
- `src/components`：跨功能复用的 UI 组件。
- `src/features`：专题馆、展厅、比较、搜索、播放器、用户足迹等业务模块。
- `src/content`：Schema、内容加载器和查询函数。
- `src/routes`：页面级组件。
- `src/styles`：设计令牌、全局样式和无障碍样式。
- `content`：版本化 JSON 内容；事实字段必须提供来源。
- `public`：经授权的分发媒体资源，不存放原始档案。
- `docs`：产品、数据与技术方案。
- `scripts`：内容校验和资源处理脚本。
- `tests/e2e`：端到端测试。

## 质量门禁

提交前按顺序执行：

```bash
npm run format:check
npm run lint
npm run typecheck
npm run test
npm run build
```

涉及路由、播放器、全景或核心互动时，还要执行 `npm run test:e2e`。若修改多个 TSX 组件，需要检查组件职责、Hooks 依赖、无障碍名称、加载与错误状态。

## 内容与无障碍

- 每个展厅必须提供来源、更新时间和权利信息。
- 图片必须有合适的替代文本；音视频必须有字幕或文字稿。
- 核心操作支持键盘，触控目标不小于约 44×44 CSS px。
- 不能只靠颜色表达身份或状态。
- 动效必须尊重 `prefers-reduced-motion`。
- AR、3D、全景等增强体验必须提供可用的图文或视频降级。

## Git 与发布

- 每个里程碑使用独立的 `codex/` 分支。
- 提交信息简洁描述完整变更，提交前不得混入无关文件。
- `main` 是 GitHub Pages 正式发布分支；合并后必须执行远端核验。
- 不提交 `node_modules/`、`dist/`、测试报告或本地环境文件。
